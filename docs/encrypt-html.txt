HTML Encryption — Setup & Usage
=================================

Scripts: ~/Script/encrypt-html.py, ~/Script/decrypt.js, ~/Script/gen-passphrase.sh
Scheme:  AES-256-GCM + PBKDF2-SHA256 (260,000 iterations)

SETUP (one-time)
----------------
1. Generate a passphrase — write it down, store in multiple safe places:
   gen-passphrase.sh          # 12 words ~154 bits entropy (default)
   gen-passphrase.sh 6        # 6 words  ~77 bits entropy

   Example: cattle finesse grannies fragiler races shilling rewiring lowbrow biology cutups gearbox vault

   Words are space-separated. 12 words is effectively unbreakable including
   against quantum attacks (Grover's algorithm halves entropy to ~77 bits —
   still beyond reach).

   WARNING: There is no recovery if the passphrase is lost.
   The same passphrase encrypts all documents — keep it consistent.

ENCRYPT
-------
# Encrypt a single file
encrypt-html.py file.html output/

# Encrypt entire html directory
encrypt-html.py ~/Documents/html/ ~/Documents/html-encrypted/

# Encrypt with passphrase from env (for scripting)
ENCRYPT_PASSPHRASE="your-words-here" encrypt-html.py ~/Documents/html/ ~/Documents/html-encrypted/

Prompts for passphrase twice if not set via env var.
Each file gets a unique random salt and IV — identical inputs produce different ciphertexts.

DECRYPT IN BROWSER
------------------
1. Open encrypted .html file in browser
2. Open browser console:  F12 → Console  (or Ctrl-Shift-J)
3. Paste the entire contents of ~/Script/decrypt.js
4. Press Enter
5. Type passphrase into the prompt() dialog
6. Page renders in clear text — passphrase never leaves the browser

AUTHORSHIP
----------
The encrypted corpus on the public web + possession of the passphrase
establishes authorship. Anyone can verify: "this was encrypted with key X,
and X was known only to the author at time of publication."

Each encrypted file contains in its metadata:
  <meta name="encryption" content="AES-256-GCM/PBKDF2-SHA256/260000">
  <meta name="source" content="original-filename.html">
  <title>Original Title [encrypted]</title>

WHAT IS STORED IN EACH FILE
----------------------------
data-salt   32-byte random salt (unique per file, public)
data-iv     12-byte random IV   (unique per file, public)
data-ct     AES-256-GCM ciphertext + 16-byte auth tag (base64)
data-iter   PBKDF2 iteration count (public)

The salt and IV are safe to publish — without the passphrase they reveal nothing.
The GCM auth tag detects any tampering with the ciphertext.
