Web Server Setup — dpcpbp.org
OpenBSD 7.8 / httpd / Let's Encrypt
=====================================

Files staged at: ~/webserver-setup/ on v2 (claude@96.30.205.85)


ORDER OF OPERATIONS
-------------------

1. CLOUDFLARE DNS
   Add two A records for dpcpbp.org:
     Type  Name   Content        Proxy
     A     @      96.30.205.85   gray cloud (DNS only)
     A     www    96.30.205.85   gray cloud (DNS only)

   Use gray cloud (DNS only) until certs are issued.
   You can enable orange cloud (CDN/proxy) after TLS is working.

2. WAIT FOR DNS PROPAGATION
   Check with: host dpcpbp.org
   Or: dig dpcpbp.org @1.1.1.1

3. OPEN FIREWALL (as root on v2)
   Edit /etc/pf.conf — add from pf-web.snippet:
     pass in  proto tcp from any to any port { 80, 443 }
     pass out proto tcp from any to any port { 80, 443 }
   Then: pfctl -f /etc/pf.conf

4. INSTALL CONFIG FILES (as root on v2)
   cp ~/webserver-setup/httpd.conf       /etc/httpd.conf
   cp ~/webserver-setup/acme-client.conf /etc/acme-client.conf
   mkdir -p /var/www/htdocs/dpcpbp.org
   mkdir -p /var/www/acme
   chown www:www /var/www/htdocs/dpcpbp.org /var/www/acme
   mkdir -p /etc/ssl/private
   chmod 700 /etc/ssl/private

   Or just run: sh ~/webserver-setup/setup-webserver.sh

5. START HTTPD (HTTP only first)
   rcctl enable httpd
   rcctl start httpd

6. ISSUE TLS CERTIFICATE
   acme-client -v dpcpbp.org
   (DNS must be pointing to the server before this step)

7. ACTIVATE HTTPS
   rcctl reload httpd

8. CERT RENEWAL (add to root crontab: crontab -e)
   0 3 * * * acme-client dpcpbp.org && rcctl reload httpd


FILE LOCATIONS ON SERVER
------------------------
  /etc/httpd.conf              — web server config
  /etc/acme-client.conf        — Let's Encrypt config
  /etc/pf.conf                 — firewall (edit manually)
  /etc/ssl/dpcpbp.org.fullchain.pem  — TLS certificate
  /etc/ssl/private/dpcpbp.org.key    — TLS private key
  /var/www/htdocs/dpcpbp.org/        — web root (put HTML/PDF here)
  /var/www/acme/               — ACME challenge dir (managed automatically)
  /var/www/logs/access.log     — access log
  /var/www/logs/error.log      — error log


NOTES
-----
- httpd runs chrooted to /var/www by default (OpenBSD security feature)
- Paths inside httpd.conf are relative to /var/www (e.g. root "/htdocs/dpcpbp.org")
- Large files (PDF etc.) supported — 200MB request body limit set in httpd.conf
- After enabling Cloudflare orange cloud proxy, update pf if needed (Cloudflare IPs only)
- root SSH key for v2 is not on this machine; copy files or sudo from claude account
